# Comunicación entre equipos

Carpeta para los pendientes que un equipo le pide al otro. Cada documento describe qué falta, por qué, y el shape/contrato esperado.

## Estructura

- `pendientes-backend/`: cosas que el **backend** debe implementar, solicitadas por el frontend (endpoints, campos, reglas).
- `pendientes-frontend/`: cosas que el **frontend** debe implementar, solicitadas por el backend.

## Regla

El equipo que pide algo lo documenta en la carpeta del equipo que lo tiene que hacer. Ejemplo: el frontend necesita un endpoint nuevo, lo escribe en `pendientes-backend/`.
